# 反模式与违规行为完整清单

本文档列出项目中定义的所有反模式和违规行为，供审查 Agent 逐条检查。评分标准已全面收紧，扣分理由必须具体到句子/段落级别。

## 一票否决行为（直接判定 overall_score < 65）

| 编号 | 行为 | 检测方法 | 示例 |
|---|---|---|---|
| VETO-01 | "待实时确认"滥用 | 标注"待确认"超过 3 处且均无途经细节/过夜建议/分段方案 | "漠河→海拉尔约600km（待实时确认）"——无满归/根河途经建议 |
| VETO-02 | 核心路线超 30% 无实质内容 | 用户请求的主干路线中，超过 30% 路段只有起终点+总距离 | 环中国方案中"第6阶段1500km"只写了4个地点名，无任何可执行细节 |
| VETO-03 | 关键决策无论证 | 路线方向/季节安排/预算分配只给结论不给推理 | "选择顺时针"但未说明为什么不是逆时针 |
| VETO-04 | 需求追问被跳过 | 首次请求且用户未说"别问"但直接输出完整方案 | 用户说"帮我规划"就直接出方案，未确认目的地/天数/起点 |

## 一、数据真实性违规（critical）

| 编号 | 违规行为 | 检测方法 |
|---|---|---|
| V-01 | 编造地理数据（坐标、POI ID、距离、用时、评分、人均、营业信息） | answer 中有具体数值但 route_validation 无对应验证记录 |
| V-02 | 编造公交线路号 | answer 中出现具体线路号但无 amap_route_transit 验证记录 |
| V-03 | 编造等待时间 | answer 中出现具体等待分钟数但无工具返回依据 |
| V-04 | 将知乎内容当作地理事实 | answer 中引用攻略内容但未标注为"攻略/主观信号" |
| V-05 | 声称"已验证"但无工具调用证据 | thinking_result 或 planning_process 中出现"已验证"但 route_validation 为空 |
| V-06 | 用模糊措辞伪装未验证事实 | answer 中出现"应该""大概""一般""通常" + 具体数据 |

## 二、工作流违规（critical）

| 编号 | 违规行为 | 检测方法 |
|---|---|---|
| V-07 | 跳过追问直接给方案 | 首次请求且用户未说"别问"，但 insufficient_information 为 false |
| V-08 | 追问后继续追问 | 用户回复第一轮追问后，再次输出 insufficient_information: true |
| V-09 | 追问未覆盖所有必须缺口 | 追问中缺少目的地/时间/起点/规划目标中的任何一项 |
| V-10 | 先写行程再验证路线 | planning_process 中路线验证步骤在行程设计之后 |
| V-11 | 跳过攻略素材采集 | 目的地明确但未调用 zhihu_guide_material |
| V-12 | 跳过地理事实验证 | answer 中有具体距离/时间数据但未委托 amap-agent |
| V-13 | 跳过审查步骤 | 最终输出未经 review-agent 审查 |
| V-14 | 输出非 JSON 格式 | 输出了 markdown 代码块、前后缀说明或额外文本 |

## 三、规划质量违规（major）

| 编号 | 违规行为 | 检测方法 |
|---|---|---|
| V-15 | 一天内跨区跳跃 | 一天内的主停留点分布在不相邻的区县或区域 |
| V-16 | 主停留点过多 | 一天超过 5 个主停留点且用户未要求紧凑 |
| V-17 | 强制热门 POI 进不顺路路线 | 热门景点被安排在明显绕路的位置 |
| V-18 | 过度推荐拥挤/商业化/网红化场景 | answer 中大量推荐高流量景点且无替代方案 |
| V-19 | 无差异化本地体验 | 所有推荐都是大众热门景点，无本地街区/市场/公园等 |
| V-20 | 未标注拥挤风险 | 包含高流量 POI 但未说明拥挤风险或提供替代 |
| V-21 | 未加入缓冲时间 | 路线安排过紧，未考虑用餐/休息/拍照/排队/交通缓冲 |
| V-22 | 每天无清晰主题 | 行程安排没有明确的路线主题（如老街散步、博物馆半日） |
| V-23 | 内容热度优先于路线可行性 | 知乎推荐的地点明显不顺路但被保留 |
| V-24 | 按热度而非慢旅游适配度评分 | 评分逻辑偏向热门度而非 route_smoothness / slow_travel_fit |

## 四、输出格式违规（major）

| 编号 | 违规行为 | 检测方法 |
|---|---|---|
| V-25 | 未使用小标题范式 | 主停留点未使用 `### 地点名` 格式 |
| V-26 | 停留点信息不完整 | 缺少距离/推荐交通/最多等待/路程时间/为什么推荐/简单介绍中的任何一项 |
| V-27 | 未区分事实与观点 | answer 中未区分"高德已确认"和"攻略/主观信号" |
| V-28 | 地点介绍过长 | 单个地点介绍超过 8 行 |
| V-29 | 堆砌表格 | 未被用户要求时使用大量表格 |
| V-30 | 内容过短敷衍 | answer 长度明显不足（如 3 天行程不足 500 字） |

## 五、思考质量违规（major）

| 编号 | 违规行为 | 检测方法 |
|---|---|---|
| V-31 | thinking_result 缺少需求识别 | 未提及用户的目的地/天数/交通偏好等关键需求 |
| V-32 | thinking_result 缺少信息缺口判断 | 未分析信息充足性 |
| V-33 | thinking_result 缺少工具调用依据 | 调用了工具但未说明为什么调用 |
| V-34 | thinking_result 缺少路线取舍 | 只列了选择结果，未说明排除理由 |
| V-35 | 暴露隐藏推理链路 | thinking_result 中出现内部实现细节或逐字推理 |
| V-36 | thinking_result 与 planning_process 高度重复 | 两者内容几乎相同，无区分 |

## 六、内容深度违规（major/critical）

> 形式合规不等于内容合格。以下问题不会被格式检查捕获，但严重影响回答的实际价值。扣分力度已收紧。

| 编号 | 违规行为 | 检测方法 | 错误示例 | 正确示例 |
|---|---|---|---|---|
| D-01 | 地点介绍只有一句定义式描述 | "简单介绍"不足 20 字或只是"XX是XX的XX"句式 | "漠河北极村是中国最北点" | "北极村商业化较重，北红村更原始——俄罗斯族木刻楞房屋、没有路灯的夜晚能看到银河" |
| D-02 | 推荐理由与用户需求无关 | 推荐理由中未引用用户的任何特定偏好 | "自然风光壮美，值得一去" | "你说喜欢自然多一点，阿尔山火山天池是草原行程中难得的地质变化，不至于连续看一个月草原审美疲劳" |
| D-03 | 时间安排只给范围不给方案 | 多天行程只说"建议X-Y天"但不逐天分配 | "喀什建议停留3-5天" | "喀什4天：第1天老城+茶馆；第2天大巴扎+牛羊巴扎(周日)；第3天香妃园+高台民居；第4天休整补给" |
| D-04 | 路线只有起终点无途经细节 | 路线描述只有"A→B，约Xkm"但无途经点 | "漠河→海拉尔约600km（待实时确认）" | "漠河→海拉尔约600km：经满归→根河→额尔古纳，分2-3天。满归有加油站和旅馆，根河冷极纪念碑可停留1h" |
| D-05 | 预算只给总数无分项 | 预算只说"约X万"但不分项 | "10万预算可行" | "油费33,600+过路费5,000+住宿24,000+餐饮21,900+门票8,000+保养5,000+应急2,500=100,000" |
| D-06 | 无备选方案 | 长途路线无 Plan B | 独库公路方案无封路替代 | "如独库封路，绕行G218伊宁→巴音布鲁克，多200km但风景类似" |
| D-07 | 风险提示泛泛而谈 | 只说"注意XX"但不具体 | "注意高反" | "叶城→三十里营房一天从1300m升至近5000m，高反风险极大；三十里营房3700m必须过夜适应" |
| D-08 | 推荐理由雷同 | 多个地点使用相同句式 | "XX是中国最美的XX之一"×10 | 每个推荐理由结合用户特定需求和该地点独特价值 |
| D-09 | 未解释路线方向逻辑 | 长途路线未说明方向选择理由 | "选择顺时针" | "6月出发先走东北(夏季最佳)，8-9月到新疆(独库窗口)，10-11月走西藏(雪前最后窗口)，冬季在南方(避寒)" |
| D-10 | 季节性窗口无论证 | 多季节行程未论证月份选择 | "9月去北疆" | "9月独库全线开通且避开7-8月旺季人流，喀纳斯秋色初现，额济纳胡杨林变黄" |
| D-11 | "待实时确认"无补充内容 | 标注"待确认"但未提供途经/过夜/分段建议 | "约600km（待实时确认）" | "约600km（待实时确认）：建议经A→B→C分2天，B有加油站和旅馆" |
| D-12 | 无逐阶段预算分配 | 有总预算分项但未按阶段拆分 | 总表有油费33,600但不分阶段 | 第1-3阶段(东北)油费约4,000；第4-6阶段(新疆/西藏)油费约12,000... |
| D-13 | 无城市休整点标注 | 长途路线未标注补给/休整节点 | 全程无"城市休整点"标注 | "城市休整点：丹东、延吉、黑河（地级市，设施齐全）" |

## 七、偷懒行为（minor/major）

| 编号 | 偷懒行为 | 严重程度 | 检测方法 |
|---|---|---|---|
| L-01 | 跳过追问直接给方案 | critical | 见 V-07 |
| L-02 | 用模糊措辞替代具体数据 | major | answer 中出现"不远""很近""很快" |
| L-03 | 遗漏关键维度（交通/预算/人群/拥挤/备选） | major | answer 中缺少这些维度的讨论 |
| L-04 | 内容过短敷衍 | major | 见 V-30 |
| L-05 | 声称信息不足但未列问题 | critical | insufficient_information=true 但 follow_up_questions 为空 |
| L-06 | 路线验证为空但有具体数据 | critical | route_validation 为空但 answer 有具体距离/时间 |
| L-07 | 模板化填充 | major | 所有地点使用完全相同的句式结构 |
| L-08 | 照搬热门清单 | major | 直接堆叠热门景点，无差异化分析 |
| L-09 | 跳过审查步骤 | critical | 见 V-13 |
| L-10 | 先写行程再验证 | critical | 见 V-10 |
